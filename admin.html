🔥 Firebase Setup Guide

Complete these steps once to connect your admin panel to Firebase Firestore.

Create Firebase Project

Go to console.firebase.google.com → Click "Add Project" → Name it shivar-farmfresh → Continue → Disable Google Analytics (optional) → Create Project.

Get Firebase Config

In Firebase Console → Project Settings (gear icon) → General → scroll to "Your apps" → Click Web icon (</>) → Register app as shivar-admin → Copy the firebaseConfig object.

Paste Config in admin.html

Open admin/admin.html → Find the block labeled 🔥 PASTE YOUR FIREBASE CONFIG HERE → Replace the placeholder values with your actual config keys.

const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "shivar-farmfresh.firebaseapp.com",
  projectId: "shivar-farmfresh",
  storageBucket: "shivar-farmfresh.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123...:web:abc..."
};

Enable Firestore Database

Firebase Console → Build → Firestore Database → Create database → Start in test mode (for now) → Choose region (asia-south1 for India) → Done.

Enable Authentication

Firebase Console → Build → Authentication → Get started → Sign-in method → Email/Password → Enable → Save.

Then go to Users tab → Add user → Enter your admin email and password.

Set Firestore Security Rules

Firebase Console → Firestore → Rules → Replace with:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Only authenticated admin can write
    match /products/{id} {
      allow read: if true;       // public can read products
      allow write: if request.auth != null;
    }
    match /orders/{id} {
      allow read, write: if true; // orders from website
    }
    match /reviews/{id} {
      allow read: if true;
      allow write: if request.auth != null;
    }
  }
}

Connect index.html to Firebase

Add this script at the bottom of your index.html before </body> — replace config with yours:

<script type="module">
import { initializeApp } from
  "https://www.gstatic.com/firebasejs/10.12.0/firebase-app.js";
import { getFirestore, collection, getDocs }
  from "https://www.gstatic.com/firebasejs/10.12.0/firebase-firestore.js";

const firebaseConfig = { /* YOUR CONFIG */ };
const app = initializeApp(firebaseConfig);
const db = getFirestore(app);

// Load products dynamically
async function loadFirebaseProducts() {
  const snap = await getDocs(collection(db, "products"));
  if (snap.empty) return; // fall back to hardcoded
  ALL_PRODUCTS = [];
  snap.forEach(d => ALL_PRODUCTS.push({id: d.id, ...d.data()}));
  renderShop(); // re-render shop section
}
loadFirebaseProducts();
</script>

🔒 How to Hide Admin Panel from Website

The admin panel at /admin/admin.html is protected by Firebase Authentication — only your email+password can log in. But here are extra steps for maximum security:

Method 1 — Remove all links (Already done)

Remove the "Admin Panel" link from the footer of index.html. The page still exists but no visitor can find it. Only you know the URL.

Method 2 — Rename to a secret URL

Rename admin/admin.html to something random like admin/sf-control-9x7k.html in your GitHub repo. Update any bookmarks you have.

Method 3 — Firebase Auth Gate (Active)

The login screen is the primary protection. Firebase Auth ensures even if someone finds the URL, they cannot access any data without your email+password. All Firestore write operations require request.auth != null.

Method 4 — Add robots.txt to block Google indexing

Create robots.txt in your repo root:

User-agent: *
Disallow: /admin/

User-agent: Googlebot
Disallow: /admin/

This prevents Google from indexing your admin page.

Method 5 — Add meta noindex tag

The admin.html already includes this tag in the <head>:

<meta name="robots" content="noindex, nofollow">

Search engines will never list this page in results.
